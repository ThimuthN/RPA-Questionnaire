1. Purpose

Document Purpose

Assessment Hub combines candidate tracking, assessment delivery, results review, and lightweight internal administration in one workspace. This guide explains how internal teams should use the system consistently and confidently.

What the System Is For

  • Register and track candidates in a structured hiring workspace.
  • Upload and manage resumes and review notes.
  • Build assessments from reusable presets and add-ons.
  • Issue share links, Test IDs, and passcodes for candidates.
  • Review results, compare candidates, and export filtered views.
  • Manage internal users and role catalogs.

Who Should Use This Guide

  • HR admins: user access, role catalog, workflow standards, governance.
  • Recruiters / coordinators: candidate creation, imports, ownership, notes.
  • Hiring managers / reviewers: assessment review, candidate decisions, comparisons.
  • Internal business users: understanding statuses, exports, and candidate experience.
Assumption

This guide describes the built-in product flow visible in the repository. Exact add-on names, preset combinations, and role catalog entries may vary by environment because those are stored in the application database and can be configured by admins.

At-a-Glance Quick Start

1. Open People

Use the Candidates workspace to search, register, import, and update hiring records.

2. Prepare the candidate

Add the role, owner, resume, stage, and notes so the record is review-ready.

3. Build and send

Create an assessment, confirm the score mix totals 100, then share the link and passcode.

4. Review and decide

Use Results to review the score, integrity signals, linked workflow status, and next action.

2. Navigation

Main Areas of the System

People

Primary hiring workspace for candidates. Use it for search, filters, registration, notes, resumes, owners, milestones, and decisions.

Add-ons

Admin/configuration workspace for global assessment add-ons and reusable presets. Use it when standard assessment parts need to be maintained.

Assessments

Operational builder for creating a new candidate or general assessment and generating share credentials.

Results

Review workspace for completed assessments. Use it for filtering, comparison, exports, and linked candidate actions.

Live Sessions

Alternative entry path for launching or joining assessments outside the standard hiring link flow.

Users

Admin-only workspace for internal user creation, role assignment, and role catalog maintenance.

Best navigation model

Think of People as the hiring record, Assessments as the send/setup action, and Results as the evaluation workspace. Teams move back and forth across those three areas most often.

Typical End-to-End Workflow

Register the candidate

Create the candidate record with name, email, role, source, and HR owner.

Complete the profile

Upload the resume, add notes, confirm stage, and assign the owner responsible for follow-up.

Send a screener

Open the assessment builder, choose a preset or add-ons, calibrate score contribution, and generate access.

Candidate completes the assessment

The candidate sees the briefing, completes check-in, then works through the timed autosaving runtime.

Review the result

Use the Results workspace and detailed review screen to understand score, integrity, and linked candidate context.

Update the hiring decision

Move the candidate forward, keep the record in review, or reject/close the profile with a short decision summary.

3. Candidate Operations

How to Register and Manage Candidates

How to Register a Candidate

  • Go to People > Candidates.
  • Select Register candidate.
  • Enter Full name and Email. Both are required.
  • Choose a saved Role if available.
  • Select a Source such as LinkedIn, Referral, Job Portal, Company Website, Agency, or Other.
  • Set the Owner if an HR or recruiting owner is already known.
  • Save the candidate, then upload the resume and send the screener afterwards.
Duplicate protection

Candidate email addresses must be unique. If the email already exists, the system sends the user back to the existing candidate record instead of creating a duplicate.

How to Import Candidates from CSV

CSV import is available directly in the Candidates workspace. This is the fastest option for high-volume intake or recruiter handoff lists.

CSV Field Rules

Column Required Notes
fullName Yes Candidate display name.
email Yes Used as the unique candidate key.
phone No Optional contact number.
positionAppliedFor or role No Maps the role/position text during import.
hrOwner or owner No Assigns the candidate owner.
batchId No Useful for campaign or intake grouping.
resumeSource No Optional source labeling.
notesSummary No Optional short recruiter summary.

Import Behavior

  • The system requires a header row and at least one candidate row.
  • Existing emails are skipped rather than overwritten.
  • Success messaging tells users how many were imported and how many duplicates were skipped.
  • The workspace also supports bulk updates for owner assignment, inbox state changes, and shared notes.
4. Candidate Detail

Resumes, Notes, Stages, and Statuses

How to Manage Resumes

  • Open the candidate record.
  • Use the Resume section to upload or replace the PDF.
  • Use preview/download when reviewers need the file during evaluation.
  • If no resume is attached, the candidate detail view clearly shows it as missing.
Assumption

Resume upload depends on configured blob/file storage. If storage is unavailable in a given environment, the hiring process can still continue, but reviewers will lose resume preview/download from within the profile.

How to Add and Use Notes

The candidate detail page supports structured note types so review comments stay searchable and readable.

Screening Interview Technical Decision General
  • Select the note type before writing.
  • Use notes for factual observations, interview outcomes, and clear next steps.
  • Use the short Decision summary field for the single best summary of current state.
  • Remember that recent notes also feed into the candidate activity view.

How to Track Candidate Stages and Statuses

The product separates several concepts so teams can understand where the person is in the hiring process versus what happened in the assessment.

Status Layers

Layer What it means Where it is most visible
Candidate stage Broad workflow position such as screening or offer. Candidate record, results-linked workflow, reference tables.
Inbox / UI status Operational queue signal such as In progress or Need review. Candidates workspace and linked result actions.
Assessment status Whether a screener has not been sent, is in progress, or has a result. Candidates workspace and candidate result summary.
Result status Outcome of a submitted assessment. Results workspace and detailed review page.

Hiring Milestones

Each candidate record is created with a default milestone path: Registered, Screener, Interview, Advanced test, and Offer / decision. Milestones can be manual or platform-driven, and their status progresses through Not started, In progress, Done, or Skipped.

Practical guidance

Keep the candidate stage, inbox status, milestone state, and decision summary aligned. If those four items tell different stories, the pipeline becomes hard to trust.

5. Assessments

Creating and Sending an Assessment

How to Create and Send an Assessment

Start from Candidates or Assessments

Use Send assessment from a candidate record when the assessment should stay linked to that candidate.

Select a preset or add-on mix

Choose an active preset for speed or add individual assessment components from the library.

Review configuration

Confirm role/stack settings where required and check duration, weight, and required pass values.

Calibrate the score model

All selected components must add up to exactly 100 marks, and every selected component needs a positive contribution.

Choose integrity controls

Select Strict, Standard, or Relaxed depending on how controlled the candidate experience should be.

Generate and share access

The system generates an assessment link, Test ID, and passcode. Share those details with the candidate.

How the Assessment Builder Works

Builder Steps

Builder step What users do What to check before moving on
Select Choose a preset or configure individual add-ons from the library. Make sure the correct assessment types are selected for the role and purpose.
Customize Review selected exams and complete required config such as role or stack. Resolve all “needs setup” items before continuing.
Calibrate Set score contribution, pass target, and integrity mode. Total marks must equal 100; no selected exam should be zero-weight.
Share Copy the final access details or launch the assessment immediately. Confirm the right candidate, link, Test ID, passcode, and expected timing.

When to use Add-ons

The Add-on Library is the admin/configuration space for maintaining global add-on defaults and simple presets. Recruiters and reviewers usually do not need to work there daily; they use the builder output rather than editing the library itself.

6. Candidate Test Experience

Sharing, Candidate Experience, and Integrity

How Assessment Sharing Works

What is generated

  • Assessment link
  • Test ID
  • Passcode
  • Launch/open action for immediate start

Operational defaults visible in code

  • Builder-generated candidate assessments use a passcode.
  • Default maximum attempts is one.
  • Linked candidate assessments use the hiring context.
  • Overall pass target defaults to 60% unless changed.

What Candidates See Before the Test

Assessment briefing page

The candidate sees estimated duration, exam mix, pass target, autosave status, and integrity mode.

Check-in page

The candidate confirms full name, email, phone (optional), and passcode when one is required.

Availability validation

The product shows candidate-safe messages if the link is expired, already used, or otherwise unavailable.

What Candidates See During the Test

  • A timed runtime with autosave and visible progress.
  • A trust banner showing the integrity preset and latest sync message.
  • A navigator for jumping between questions.
  • Flagging, next/back movement, and section handoff between selected exams.
  • A submit review step that summarizes answered, unanswered, and flagged items before final submission.
  • A recovery modal if the session loses focus or requires a full-screen return.

Integrity / Security Modes Explained in Plain Language

Integrity Levels

Mode What the candidate experiences Best use case
Strict Protected Mode Requires full-screen, blocks copy/paste and context menu, and monitors tab switches. High-control assessments where integrity matters most.
Standard Guardrails Monitors activity and blocks clipboard/context menu, but does not force full-screen. Default hiring screeners where control is important but not overly rigid.
Relaxed Monitoring Allows normal browser behavior and keeps lightweight activity monitoring only. Low-risk or practice-style assessments.
How integrity is summarized later

Integrity review is driven by tab-hidden, copy/cut, and paste counts. In the results workspace, the risk label becomes Clean, Watch, or Review.

7. Results

How to Review Results

Results Workspace

  • Open Results to review submitted assessments across hiring and other contexts.
  • Use search and filters for result, review state, context, integrity, role, owner, stage, and score band.
  • Select up to four results for side-by-side comparison.
  • Use bulk actions to update review state or linked candidate status, owner, or notes.
  • Export the filtered view to CSV or JSON.

Detailed Result Review

The detailed result page combines scoring, interpretation, and candidate workflow context.

Top strength Main concern Confidence Integrity summary

Reviewers can also see category breakdowns, question/section review details, recent candidate activity, and quick actions to move the candidate forward, keep them in review, or reject them.

Review States

  • Unreviewed: no reviewer disposition applied yet.
  • Reviewed: evaluated and considered complete.
  • Flagged: requires additional attention or escalation.

Score Bands

  • High: 75 and above
  • Mid: 50 to 74.9
  • Low: below 50

How to Interpret Pass / Review / Fail

Outcome Interpretation

Outcome Plain-language meaning Recommended HR action
Pass The candidate met the assessment threshold and is not in the borderline band. Move forward if role fit, resume, and interview feedback also support progression.
Review The result is borderline or has enough integrity concern to deserve human review. Open the detailed result, inspect weak areas and integrity notes, then decide manually.
Fail The candidate did not meet the required threshold. Usually reject or hold, unless a manager wants a manual exception based on broader evidence.
Important nuance

A strong raw score is not the only signal. Confidence decreases when integrity activity is high or when a result is close to the pass boundary, so reviewers should read the detailed result page before making final decisions.

8. Analysis Tools

Filters, Comparison, Exports, and Admin Access

How to Use Filters, Comparison, and Exports

  • Filter the results list first, then export. Exports follow the current view.
  • Use Review needed as a fast saved action when triaging borderline or flagged items.
  • Use compare for shortlists or calibration discussions; the UI allows up to four items at once.
  • Use linked owner and stage filters when you want to isolate results tied to a specific recruiter pipeline.

How to Manage Internal Users and Roles

The Users page is admin-only. It combines two different administration tasks: managing internal application users and maintaining the role catalog used during candidate registration.

9. Support

Troubleshooting / FAQ

“The user cannot log in.” Confirm the email/password combination and whether the user is a database-backed user or bootstrap admin. Only admins can open the Users workspace.

“The candidate already exists.” The system blocks duplicate emails. Open the existing candidate instead of creating another record.

“My CSV import failed.” Make sure the file has a header row and includes fullName and email. Duplicate emails are skipped, not overwritten.

“The candidate says the test link is unavailable.” Common causes are expired access, a used single-attempt link, or a missing/incorrect passcode.

“The candidate cannot continue after switching tabs.” Depending on integrity mode, the runtime may require resuming focus or re-entering full-screen before continuing.

“We see a save warning during the test.” The runtime is designed to autosave regularly. Ask the candidate to keep the tab open and let the connection re-stabilize before closing the session.

“No result appears in the hiring workflow.” Confirm the assessment was linked to a candidate at send time; otherwise the result may exist as an assessment-only record.

“Why is the candidate marked Need review?” That status is used when a submitted assessment result needs human decisioning, especially around borderline or reviewed outcomes.

“Why is a role missing from registration?” It may be inactive in the role catalog. An admin can reactivate or recreate it from Users.

Best Practices

  • Use one consistent owner naming format so filters and reporting stay clean.
  • Complete the resume and short decision summary before escalating a candidate to a hiring manager.
  • Prefer linked candidate assessments over general assessments when the result should drive a hiring workflow.
  • Review integrity signals together with score, not in isolation.
  • Use note types deliberately so screening, technical, and decision comments remain easy to scan.
  • Deactivate old roles instead of deleting them, so historical records stay understandable.
  • Export filtered result sets for meetings instead of exporting everything and sorting externally.
  • Use compare view for final shortlist calibration instead of relying on memory or disconnected spreadsheets.
